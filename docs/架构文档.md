## 2. 核心数据流

### 2.1 实时监控流程

```
[timer触发] → [scheduler.py]
                  │
                  ▼
         [realtime_monitor()]
                  │
     ┌────────────┼────────────┐
     ▼            ▼            ▼
[market_data]  [fund_flow]  [market_overview]
  拉行情         拉资金        拉市场概览
     │            │            │
     └────────────┼────────────┘
                  ▼
          [evaluate_realtime_rules()]
           遍历规则 → 匹配条件
                  │
                  ▼
          [process_alerts()]
           去重→排序→限频→存储
                  │
                  ▼
          [send_alert_batch()]
              飞书推送
```

### 2.2 新闻采集流程

```
[timer触发] → [news_collector()]
                  │
     ┌────────────┼────────────┐
     ▼            ▼            ▼
[财联社电报]  [东方财富]  [新浪财经]
     │            │            │
     └────────────┼────────────┘
                  ▼
          [fetch_all_news()]
           合并 + 去重
                  │
                  ▼
          [evaluate_news_alerts()]
           情绪分析 + 关键词匹配
                  │
                  ▼
          [process_alerts()] → [推送]
```

### 2.3 盘后总结流程

```
[15:10 cron触发] → [post_market_summary()]
                        │
     ┌──────────────────┼──────────────────┐
     ▼                  ▼                  ▼
[拉收盘行情]     [拉指数数据]      [拉当日新闻]
     │                  │                  │
     └──────────────────┼──────────────────┘
                        ▼
            [generate_daily_summary()]
              统计涨跌/涨停/跌停
                        │
           ┌────────────┼────────────┐
           ▼                         ▼
    [send_daily_summary()]   [send_daily_summary_email()]
        飞书推送                   邮件推送
```

## 4. 数据模型

### 4.1 SQLite表结构

```sql
-- 行情快照表
stock_snapshot (
    id, code, name, price, change_pct, volume, amount,
    turnover, high, low, open, pre_close, volume_ratio, fetched_at
)

-- 每日总结表
daily_summary (
    id, date, sh_index, sh_change_pct, sz_index, sz_change_pct,
    cyb_index, cyb_change_pct, kcb_index, kcb_change_pct,
    up_count, down_count, limit_up_count, limit_down_count,
    total_amount, created_at
)

-- 告警日志表
alert_log (
    id, alert_at, alert_type, category, stock_code, stock_name,
    rule_id, title, detail, notified, created_at
)

-- 新闻缓存表
news_cache (
    id, source, title, content, url, pub_time, fetched_at
)
```

### 4.2 内存状态模型

```python
class AppState:
    running: bool                    # 运行状态
    start_time: datetime             # 启动时间
    today_is_trading_day: bool       # 今日是否交易日
    market_session: str              # 当前市场时段
    quote_cache: dict                # {code: snapshot}
    intraday_extremes: dict          # {code: {high, low, open}}
    news_cache: list                 # 新闻列表
    alert_cooldowns: dict            # {code:rule_id: last_time}
    today_stats: dict                # 今日统计
```

## 5. 配置管理

```
config/
├── settings.yaml       # 主配置
│   ├── notify          # 通知配置（飞书/邮件）
│   ├── scheduler       # 调度配置（间隔/时间）
│   ├── analysis        # 分析阈值
│   ├── data_source     # 数据源配置
│   ├── logging         # 日志配置
│   └── web             # Web服务配置
├── watchlist.yaml      # 自选股+指数
│   ├── stocks[]        # 股票列表
│   └── indices[]       # 指数列表
└── alert_rules.yaml    # 告警规则
    └── rules[]         # 规则列表
```

## 7. 安全与稳定性

### 7.1 数据安全
- 配置文件不包含在版本管理中（通过.gitignore排除settings.yaml）
- Webhook URL和SMTP密码等敏感信息存放在本地config中

### 7.2 容错机制
- 数据源异常时不崩溃，返回空数据
- 飞书推送失败不影响后续监控
- 告警冷却防止重复推送

### 7.3 性能考虑
- 内存缓存减少重复API调用
- SQLite轻量存储适合单机运行
- 批量写入减少数据库IO
