# 自动盯盘Agent — 需求文档

## 1. 项目概述

### 1.1 项目背景
A股投资者在日常工作中难以持续盯盘，错过买卖时机。需要一个自动化系统实时监控市场动态，在出现机会和风险时主动推送通知，同时整合国内外重要财经新闻，帮助投资者做出更及时的决策。

### 1.2 目标用户
- **主要用户**：程序员+股民（需要兼顾工作和投资）
- **使用场景**：工作日内无法持续看盘，需要被动接收关键信息

### 1.3 核心目标
- 7×24小时运行（交易时段重点监控）
- 实时行情异动检测，延迟 < 30秒
- 多渠道推送（飞书 + 邮件）
- 整合国内外财经新闻
- 可配置的自选股和告警规则

## 3. 非功能需求

### 3.1 性能
- 实时行情刷新延迟 < 30秒
- 告警从触发到推送 < 10秒
- Web看板响应 < 500ms

### 3.2 可靠性
- 数据源异常时降级处理，不崩溃
- 飞书推送失败不影响主流程
- 自动重连机制

### 3.3 可维护性
- 配置与代码分离（YAML）
- 日志分级记录
- 模块化架构

### 3.4 可扩展性
- 数据源可替换（预留接口）
- 告警规则可配置
- 推送渠道可扩展

## 5. 告警规则详情

### 5.1 风险类告警（优先级高）
| 规则 | 触发条件 | 默认阈值 |
|------|----------|----------|
| 快速下跌 | 跌幅超过阈值 | -5% |
| 跌停 | 触及跌停板 | -9.8% |
| 放量下跌 | 量比>1.5 且 跌幅>2% | 见规则 |
| 冲高回落 | 最高涨>5% 后回落>3% | 见规则 |
| 主力资金流出 | 主力净流出超1亿 | 1亿 |
| 行业利空 | 新闻含利空关键词 | 见规则 |

### 5.2 机会类告警
| 规则 | 触发条件 | 默认阈值 |
|------|----------|----------|
| 快速拉升 | 涨幅超过阈值 | +5% |
| 放量上涨 | 量比>2 且 涨幅>=0 | 见规则 |
| 主力资金流入 | 主力净流入超1亿 | 1亿 |
| 行业利好 | 新闻含利好消息关键词 | 见规则 |

### 5.3 信息类告警
| 规则 | 触发条件 | 默认阈值 |
|------|----------|----------|
| 涨停 | 触及涨停板 | +9.8% |

## 6. 用户操作流程

### 6.1 初次使用
1. 安装Python 3.11+ 和依赖
2. 配置飞书Webhook URL
3. 修改自选股列表
4. 调整告警规则阈值（可选）
5. 启动系统

### 6.2 日常使用
1. 系统自动在交易时段监控
2. 飞书实时推送告警
3. 盘前收到简报
4. 盘后收到总结
5. 可通过Web看板查看实时状态
